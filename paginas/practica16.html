16. Subconsultas (Subqueries)

Una subconsulta es una consulta SELECT anidada dentro de otra instrucción SQL (como otro SELECT, INSERT, UPDATE o DELETE).

Niveles de Anidación (Select sobre Select)

Las subconsultas se estructuran en diferentes niveles según qué tan profundo sea el anidamiento:

  • 2 Niveles (SELECT sobre SELECT): Una consulta que contiene una subconsulta simple en su cláusula WHERE o FROM.
  • 3 Niveles (SELECT sobre SELECT sobre SELECT): Una consulta principal que contiene una subconsulta, la cual a su vez contiene otra subconsulta interna.
1Subconsulta Básica (2 Niveles)

Obtiene los empleados que ganan más que el salario promedio de la empresa:

SELECT nombre, salario
            FROM empleados
            WHERE salario > (
                SELECT AVG(salario) 
                FROM empleados
            );
2Departamento con Menor Salario

Obtiene el nombre del departamento que tiene al empleado con el salario más bajo:

SELECT d.nombre
            FROM departamentos d
            INNER JOIN empleados e ON d.id = e.depto_id
            ORDER BY e.salario ASC
            LIMIT 1;
3Filtrar Departamentos con IN

Selecciona departamento y empleado para las áreas de Ventas o TI:

SELECT d.nombre, e.nombre
            FROM empleados e 
            INNER JOIN departamentos d ON d.id = e.depto_id
            WHERE d.nombre IN ('ventas', 'TI');
4Subconsulta Anidada (3 Niveles)

Obtiene los empleados del departamento que tiene el presupuesto máximo asignado (3 SELECTs anidados):

SELECT nombre, salario 
            FROM empleados 
            WHERE depto_id = (
                SELECT id 
                FROM departamentos 
                WHERE presupuesto = (
                    SELECT MAX(presupuesto) 
                    FROM departamentos
                )
            );
