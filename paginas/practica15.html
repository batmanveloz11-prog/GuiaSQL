JOIN - INNER JOIN

Un INNER JOIN se utiliza para combinar filas de dos o más tablas en función de una columna relacionada entre ellas, devolviendo registros únicamente cuando existe una coincidencia en ambas tablas.

Estructura de las Tablas (clientes y pedidos) 
CREATE TABLE clientes (
                id_cliente INT AUTO_INCREMENT PRIMARY KEY,
                nombre VARCHAR(100) NOT NULL,
                email VARCHAR(100) NOT NULL UNIQUE,
                telefono VARCHAR(20),
                fecha_registro TIMESTAMP DEFAULT CURRENT_TIMESTAMP
            );
CREATE TABLE pedidos (
                id_pedido INT AUTO_INCREMENT PRIMARY KEY,
                id_cliente INT NOT NULL,
                fecha_pedido TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
                total DECIMAL(10, 2) NOT NULL,
                estado VARCHAR(50) DEFAULT 'Pendiente',
                FOREIGN KEY (id_cliente) REFERENCES clientes(id_cliente) ON DELETE CASCADE ON UPDATE CASCADE
            );

Consulta INNER JOIN

Devuelve solo los clientes que han realizado al menos un pedido:

SELECT 
                clientes.id_cliente, 
                clientes.nombre, 
                pedidos.id_pedido, 
                pedidos.total, 
                pedidos.estado
            FROM clientes 
            INNER JOIN pedidos ON clientes.id_cliente = pedidos.id_cliente;

JOIN - LEFT JOIN y RIGHT JOIN

Los Joins externos devuelven registros incluso cuando no hay coincidencias perfectas en la relación entre tablas.

1. LEFT JOIN (Consulta externa izquierda)

Devuelve todos los registros de la tabla izquierda (clientes), y las filas coincidentes de la derecha (pedidos). Si no hay coincidencia, muestra valores NULL.

SELECT 
                clientes.id_cliente, 
                clientes.nombre, 
                pedidos.id_pedido, 
                pedidos.total
            FROM clientes 
            LEFT JOIN pedidos ON clientes.id_cliente = pedidos.id_cliente;

2. RIGHT JOIN (Consulta externa derecha) Visualización Completa

Devuelve todos los registros de la tabla derecha (pedidos), y las filas coincidentes de la izquierda (clientes). Si hay algún pedido sin cliente (o si el cliente no coincide), mostrará NULL en los campos del cliente.

SELECT 
                clientes.id_cliente, 
                clientes.nombre, 
                pedidos.id_pedido, 
                pedidos.total
            FROM clientes 
            RIGHT JOIN pedidos ON clientes.id_cliente = pedidos.id_cliente;
Resultado de la consulta (Diferencia visible):
id_cliente nombre id_pedido total
1Juan Pérez1$150.50
1Juan Pérez2$45.00
2Ana Gómez3$350.00
3Carlos Ruiz4$89.90
4María López5$12.50
2Ana Gómez6$120.00
Laura Torres (ID 5) NO APARECE aquí porque no tiene ningún pedido en la tabla derecha (pedidos).
*(Solo se muestran los registros de la tabla derecha "pedidos". Al estar todos los pedidos asignados a clientes existentes, no hay valores NULL en la columna cliente).
