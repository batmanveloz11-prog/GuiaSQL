Order by, limit y top (pdf de Abraham)

ORDER BY, LIMIT y TOP (Katanas Edition)

Aprende a ordenar tus resultados y a restringir la cantidad de registros devueltos usando ORDER BY, LIMIT (MySQL) y su equivalente TOP (SQL Server).

Diferencia de Motores: TOP vs LIMIT

La palabra clave TOP pertenece a SQL Server (Microsoft). En MySQL, se utiliza LIMIT al final de la consulta para lograr el mismo resultado. Aquí verás ambos enfoques.

1Ordenar Ascendente (ORDER BY simple)

Ordena las katanas de menor a mayor precio (el orden ascendente es por defecto):

SELECT * FROM katanas
                ORDER BY precio;
2Ordenar Descendente (ORDER BY DESC)

Ordena las katanas de mayor a menor precio:

SELECT * FROM katanas
                ORDER BY precio DESC;
3Ordenar por múltiples columnas

Ordena por material (A-Z) y si se repite, por longitud (de mayor a menor):

SELECT * FROM katanas
                ORDER BY material ASC, longitud DESC;
4Limitar resultados (LIMIT en MySQL)

Obtiene únicamente los primeros 3 registros de la tabla:

SELECT * FROM katanas
                LIMIT 3;
5Paginación con Desplazamiento (LIMIT OFFSET)

Ignora los primeros 2 registros y obtiene los siguientes 3:

SELECT * FROM katanas
                LIMIT 3 OFFSET 2;
6Obtener primeros registros (TOP en SQL Server)

Equivalente de LIMIT en SQL Server (va después de la palabra SELECT):

SELECT TOP 5 * 
                FROM katanas;
7Obtener un porcentaje de filas (TOP PERCENT en SQL Server)

Obtiene el primer 25% de las filas de la tabla (no soportado directamente en MySQL nativo simple):

SELECT TOP 25 PERCENT *
                FROM katanas
                ORDER BY precio DESC;
