5. Claves Primarias, Foráneas y Normalización

Aprende a relacionar tus tablas de forma segura garantizando la integridad de tus datos y aplicando las reglas de Normalización (1FN, 2FN y 3FN) para evitar duplicidad.

Claves Primarias (Primary Key) y Foráneas (Foreign Key)

La relación entre tablas se logra mediante identificadores únicos llamados claves:

  • Clave Primaria (PK): Columna o conjunto de columnas que identifican de forma única a cada registro en una tabla. No puede contener valores nulos ni repetirse.
  • Clave Foránea (FK): Columna que crea un enlace entre dos tablas. Hace referencia a la Clave Primaria (PK) de otra tabla, impidiendo registrar datos huérfanos.
Ejemplo de código SQL de relación (Padre - Hijo):
-- 1. Tabla Padre (Independiente)
            CREATE TABLE departamentos (
                id INT AUTO_INCREMENT PRIMARY KEY,
                nombre VARCHAR(50) NOT NULL
            );
            -- 2. Tabla Hija (Dependiente de departamentos)
            CREATE TABLE empleados (
                id INT AUTO_INCREMENT PRIMARY KEY,
                nombre VARCHAR(100) NOT NULL,
                departamento_id INT,
                -- Definición de la Clave Foránea
                FOREIGN KEY (departamento_id) REFERENCES departamentos(id)
                    ON DELETE SET NULL
                    ON UPDATE CASCADE
            );
Acciones Referenciales: ON DELETE SET NULL pone en nulo el campo si se borra el departamento relacionado. ON UPDATE CASCADE actualiza el ID en cascada en la tabla empleados si el ID del departamento cambia.
Normalización de Bases de Datos

La normalización es el proceso de organizar las columnas y tablas para **reducir la redundancia (datos duplicados)** y asegurar que las dependencias tengan sentido lógico.

Primera Forma Normal (1FN): Atomicidad

Elimina grupos repetidos o listas de datos en una sola celda. Cada celda debe contener un único valor indivisible (atómico).

Segunda Forma Normal (2FN): Dependencia Total

Debe estar en 1FN. Toda columna que no sea clave debe depender por completo de la clave primaria. Se eliminan las dependencias parciales separando los datos en diferentes tablas.

Tercera Forma Normal (3FN): Sin Dependencias Transitivas

Debe estar en 2FN. Ninguna columna debe depender de otra columna que no sea la clave primaria (los atributos no clave deben ser independientes entre sí).
